# Task 08 ‚Äî UI

## Scope

Build the actual Blade views: the Import tab's upload form, the "Import settings"
card (size cap + background toggle), and the in-progress-imports list with
Resume/Discard actions. **Does not** change any controller/service logic ‚Äî this task
only wires views to the data/routes tasks 06‚Äì07 already expose.

* `resources/views/admin/data/index.blade.php`: replace the `#panel-import`
  placeholder with the upload form from `ui.md` ‚Üí *Where it lives*; add the "Import
  settings" card from `ui.md` ‚Üí *Import settings card*; add the in-progress imports
  list from `ui.md` ‚Üí *In-progress / stalled imports list*, reusing the existing
  `<x-delete-button>` component for Discard (do not hand-roll a new confirm form).
* `DataTransferController::index()` (already updated in task 06 to pass the data):
  confirm the view actually receives `$importSetting` and `$imports` and renders
  without errors.
* `app/Enums/ImportPhase::label()` (added in task 01) is what the status list
  displays ‚Äî no new label logic in the view.

## Depends on

Task 06 (routes, `ImportSetting`/`Import` data available to the view), Task 07
(the background-mode checkbox needs `run_in_background` to actually do something).

## Key decisions already made

* Reuse `<x-delete-button>` for Discard ‚Äî don't hand-write a `confirm()` form.
* The background-processing checkbox includes the "requires a running queue worker"
  caveat text from `ui.md` ‚Äî don't drop it, it's the reason the setting defaults off.
* No drag-and-drop upload, no new error-display pattern beyond `<x-input-error>` ‚Äî
  per `ui.md`'s Accessibility section.

## Docs to consult

`ui.md` (the whole document ‚Äî this task implements it close to verbatim).

## Tests

Feature-level, via `route()` + `assertSee`/`assertSessionHasErrors` (this project has
no JS/browser test suite per the existing conventions, so Alpine tab-switching isn't
covered by an automated test ‚Äî note that as a manual-verification item):

* The Import tab renders the upload form when `admin.data.index` is visited.
* A successful upload followed by the redirect shows the flash "Project imported."
  (or "Import queued." in queued mode).
* A stalled `Import` row (factory-created at `phase = story`) appears in the
  in-progress list with its phase label and Resume/Discard buttons.
* After Resume completes, the row no longer appears (phase reached `completed`).
* After Discard, the row and its partially-created project are both gone.
* The Import settings form pre-fills with the current `ImportSetting` values and
  round-trips a change through `admin.data.import-settings`.
